Анализ проекта ClearEditor — рекомендации по улучшению
======================================================

1. ТЕСТЫ
   - Нет тестового фреймворка. Все три теста — ручные визуальные скрипты.
   - Добавить pytest и покрыть core/* unit-тестами (без QApplication, через mock).
   - Тесты не запускаются одной командой — нет точки входа (pytest, unittest, tox).

2. ЛИНТЕР / ФОРМАТТЕР / ТИПИЗАЦИЯ
   - Нет ни одного инструмента: ruff, black, mypy, flake8 — ничего.
   - Смешанные отступы: msg_panel.py на 4 пробелах, весь остальной код на табах.
   - Нет pyproject.toml — негде описать тулы и конфигурацию.

3. README.MD УСТАРЕЛ
   - Упоминает cleanmain (старое имя репозитория) и cleanmain.py (несуществующий файл).
   - Не описывает плагины, архитектуру, структуру папок.

4. ОБРАБОТКА ОШИБОК И КРАЙНИХ СЛУЧАЕВ
   - Save As не спрашивает подтверждение при перезаписи существующего файла.
   - Нет подтверждения на "Clear recent files" — удаляет всё без диалога.
   - При отсутствии config.json — ConfigError с русским текстом, выход.
   - Спецификации на русском, код и коммиты на английском — смешение языков.

5. ШРИФТ (ZOOM)
   - QTextEdit (ExtEditor) не наследует шрифт родителя автоматически.
     Zoom In/Out меняет шрифт EditorApp, но уже открытые вкладки не обновляются.
   - Нужно переустанавливать шрифт для currentWidget().editor при каждом zoom.

6. НЕТ CI
   - Нет GitHub Actions или другого CI для автоматической проверки кода.

7. АРХИТЕКТУРНЫЕ ЗАМЕЧАНИЯ
   - ExtEditor назван "ext" (extended), но де-факто это HTML-редактор.
   - PluginManager._setup_ui() использует self.parent() для доступа к EditorApp,
     хотя editor передаётся аргументом — путаница источников.
   - Нет init-файла в plugins/ — PyQt5 не требует, но может сбивать с толку.
   - QSettings не имеет organizationName — может влиять на расположение ini на разных ОС.

8. ИКОНКИ
   - Tests/test_*.py используют icons/clear_test.jpg и icons/clear1.jpg,
     основное приложение — icons/clear.svg.
   - Часть путей захардкожена (icons/clear.svg) — нет fallback при отсутствии файла.

9. ОБНАРУЖЕНИЕ ВНЕШНИХ ИЗМЕНЕНИЙ
   - Проверка только при фокусе окна (on_app_focus_changed).
   - Нет таймера/наблюдателя (QFileSystemWatcher).
   - Если окно остаётся в фокусе, внешние изменения не обнаруживаются.

10. ГИТ
    - Нет .gitattributes для нормализации окончаний строк.
    - Нет .venv/ в .gitignore.
    - Нет шаблона .gitignore для Python-проектов (*.pyc, .DS_Store и т.д. уже есть,
      но coverage/, .mypy_cache/, .ruff_cache/ и др. отсутствуют).
