5. PLUGIN FUNCTIONALITY REQUIREMENTS
--------------------
5.1. 'WordCount' plugin — word count in QTextEdit.
   Declaration:
     status_fields = {"word_count": {"label_template": "Word: {}",
                                     "signal_name": "word_count_changed"}}
   Signal: word_count_changed(int)
    on_load(editor): receives EditorApp, connects to ALL tabs
     editor.tab_panel (textChanged), subscribes to tab_added
     (connecting to new tabs) and currentChanged (recount on
     tab switch), triggers initial count.
   on_unload: disconnects from all textChanged, tab_added, currentChanged.
   menu_items/toolbar_items: not used.

5.2. 'TextProcessing' plugin — text processing in current tab editor.
    Declaration:
      actions — 8 items (see example in plugins_service_spec 4.8.4).
      menu_items — "Edit" menu with "HTML" submenu.
      toolbar_items — flat list: wrap_p, wrap_b, make_list.
        For compactness toolbar text is replaced with short_text
        via {**action, 'text': action['short_text']}.

    Methods:
      selectedTextProcessing(func, **kwargs) — common method,
        gets QTextCursor of selected text, replaces "\u2029" with "\n",
        applies func to text, replaces selection with result,
        restores selection of the processed block.

      remove_empty_lines_in_selected() — removes empty lines
        in selected block (lines containing only whitespace).

      capitalize_first_letters_in_selected() — capitalizes
        the first letter of each line in the selected block.

      replace_entity_in_selected() — replaces HTML entities:
        & → &amp;, < → &lt;, > → &gt;.

      wrap_p_in_selected() — wraps each non-empty line
        of the selection in <p>...</p>.

      wrap_b_in_selected() — wraps the entire selection in <b>...</b>.

      unwrap_in_selected() — removes the outer parent tag:
        finds the first <tag> and matching </tag>,
        returns the content between them.

      make_list_in_selected() — converts non-empty lines
        of the selection into a <ul>/<li> list.

      make_table_in_selected() — converts non-empty lines
        with tab separators into a <table>/<tr>/<td> table.

    on_load: saves editor.tab_panel for accessing current editor.
    on_unload: clears reference.

5.3. 'HTMLProcessing' plugin — HTML code processing in the current tab.
    Declaration:
      actions — 1 action: gen_content_list.
      menu_items — "Edit" menu with "HTML" submenu (any position,
        merges into existing HTML submenu from TextProcessing).
      toolbar_items: not used.
      Signal: message = pyqtSignal(str, str, str) — inherited from PluginBase.
        On activation: plugin.message.connect(editor_app.on_message).

    Method:
      gen_content_list(start_head_level=1, pass_nocontents=False)

      Parses the current tab's HTML content
      (editor.tab_panel.currentWidget().editor.toPlainText()),
      finds all <h1>–<h6> headings, extracts the id attribute (if present) and
      the heading text content. Builds a nested <ul>/<li> list with
      <a href="#anchor_id">heading_text</a> links.
      Outputs the result to msg_panel.view_box via setPlainText (raw HTML source).

      Parameters:
        start_head_level (int, default=1) — minimum heading level to include.
          Headings below this level are ignored.
          E.g., start_head_level=2 skips <h1>, starts from <h2>.
        pass_nocontents (bool, default=False) — if True, headings with
          class="nocontents" (<h2 class="nocontents">) are skipped.

      Generation rules:
        - id attribute: if the heading tag has an id, it is used in href.
          If no id — href="#", and a message is sent:
          self.message.emit(f'header in the {num} row has no anchor.',
                           self.__class__.__name__, 'info')
          where num is 1-indexed line number in the HTML source.

        - Nesting: lower-level headings (h2 < h3) open a new <ul>.
          Higher or equal-level headings close the current <ul> and </li>.
          On a level gap (e.g., h3 followed by h5) — an empty <ul> is inserted
          between them to maintain valid nesting.

        - nocontents class: if pass_nocontents=True, headings
          containing class="nocontents" (or class='nocontents')
          are excluded from the content list.

      Example output (View tab, via setPlainText):
        <ul>
        <li><a href="#intro">Introduction</a>
          <ul>
          <li><a href="#setup">Setup</a></li>
          <li><a href="#usage">Usage</a></li>
          </ul>
        </li>
        <li><a href="#">Basics</a></li>
        <li><a href="#advanced">Advanced topics</a>
          <ul>
          <ul>
          <li><a href="#decorators">Decorators</a></li>
          </ul>
          </ul>
        </li>
        </ul>

        Note: the empty <ul> between <li> and <li> occurs when the heading
        level jumps (e.g., h3 → h5 without h4).

    on_load: saves editor, connects plugin.message to editor.on_message.
    on_unload: clears references.
