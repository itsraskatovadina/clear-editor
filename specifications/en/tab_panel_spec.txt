6. TabPanel FUNCTIONALITY REQUIREMENTS

Tab container with tabs, managing multiple FileEditor instances.

6.1. TabPanel — QTabWidget, multi-document editor window in the main EditorApp class,
each tab contains a FileEditor. It manages the tab lifecycle (open,
close, save, reload, reorder), tracks changes across different tabs,
detects external file changes and sends aggregated state signals
to the parent main window or controller.

6.2. SIGNALS
     - tab_added = pyqtSignal(object) — emitted in add_tab() after adding a tab,
       passes the internal editor (QTextEdit/ExtEditor).
     - editor_state_changed = pyqtSignal(str, str, str, str)
       Parameters: (file_name, full_path, modification_label, operation)
       operation: 'tab_changed', 'modification_changed', 'Opened', 'New', 'Saved', 'Saved as', 'Reload', 'Reopened'
     - line_changed = pyqtSignal(int)
       Parameter: line number (1-based).
     - message = pyqtSignal(str, str, str)
       Parameters: (message_text, sender, message_type)

6.3. Tabs can be dragged (setMovable(True)) and closed (setTabsClosable(True)).

6.4. The last remaining tab cannot be closed (close_tab checks self.count() > 1), except in special cases.

6.5. INSTANCE ATTRIBUTES
     - do_not_process_the_tab_change: bool — flag to suppress tab switch handling.
     - editor_class: type — editor class (default QTextEdit).

6.6. CLASS: TabPanel(QTabWidget)
     def __init__(self, parent=None, editor_class=QTextEdit):
         super().__init__(parent)
         self.editor_class = editor_class
     - editor_class is passed to each FileEditor in a new tab.
     - Connection: currentChanged.connect(self.on_current_tab_changed)
     - Initial tab is NOT created (self.add_tab() removed). First tab
       is created via set_files() or EditorApp.set_tab_panel().

6.7. API

     6.7.1. Navigation:
            - current_editor() -> QTextEdit | None — returns the active editor of the current tab.
            - get_current_editor() -> QWidget — returns the current widget (FileEditor).
            - get_current_path() -> str: path to current editor's file or ''.

     6.7.2. Tab management:
            - add_tab(path=None) -> bool:
              If path — checks if file is already open (if so — switches to that tab).
              If path does not exist — calls check_path_exists(), which shows
              a dialog and returns False.
              Creates FileEditor, connects its signals, adds tab.
              If path is set — loads file via editor.load_from_file().
              Emits editor_state_changed with 'Opened' or 'New' operation.
            - new_tab(): wrapper for add_tab(None).
            - open_file(): QFileDialog.getOpenFileName → add_tab(Path(file_path)).
            - set_files(file_list: list[str]) -> int:
              Takes a list of file paths.
              For each path checks path.exists(), for existing calls add_tab.
              Returns number of successfully opened files.
            - close_tab(index, close_last_tab = False) -> bool:
              Last tab can only be closed if explicitly allowed.
              If document is modified — Save/Ignore/Cancel dialog.
              If file is externally modified — warning.
              On Cancel returns False.
            - close_all_tab() -> bool:
              Closes all tabs in reverse order with close_tab(), allowing last tab to be closed.
            - current_tab_close() -> bool: closes current tab.

     6.7.3. Save/load:
            - save_tab(index, editor) -> bool: saves file. If path=None — save_as_tab.
            - save_as_tab(index, editor) -> bool: QFileDialog.getSaveFileName, saves.
            - current_tab_save(): saves current tab.
            - current_tab_save_as(): saves current tab as.
            - current_tab_reload(): reloads file from disk, preserves cursor position.

     6.7.4. Information:
            - current_tab_view_property(): QMessageBox with file path.
            - get_open_files() -> list[str]: list of open file paths.
            - check_path_exists(path) -> bool: checks file existence, shows warning.
            - set_tab_text_and_tooltip(name, fname, mod_label, index): updates tab text and tooltip.

     6.7.5. Signal handlers:
            - on_current_tab_changed(index): focuses editor, emits editor_state_changed.
            - on_user_modification_changed(name, fname, mod_label): updates tab text,
              emits editor_state_changed.
            - on_app_focus_changed(old, now): checks external changes on focus return.
            - check_tab_externally_modified(index, editor): Reload/Save/Ignore dialog on external change.

     6.7.6. closeEvent(event): calls close_all_tab(), on success — event.accept().

6.8. New and Close menu items for TabPanel are in the EditorApp File menu
     (see editor_app_spec.txt).

6.9. Plugins via plugin_manager receive editor_widget = editor.current_editor().
     Plugins work with the current (active) editor regardless of which
     tab it is on. When switching tabs, the plugin continues to work
     with the new active editor (subscribes to TabPanel's currentChanged signal).
     To connect to new tabs, plugins subscribe to the tab_added signal.
