Спецификация пакета iclear
============================

1. Назначение
-------------
Пакет iclear предоставляет инструменты для навигации и работы со статическим
PHP-сайтом itsclear.ru. Состоит из двух модулей:

  - iclearlib.py — библиотека классов для представления файловой структуры
    сайта (дерево узлов), навигации, генерации index.php, aside.php,
    проверки целостности карты сайта (map.xml).
  - wiclear.py — набор PyQt5-виджетов для визуального выбора страницы
    или директории сайта и выполнения операций.

2. Файловая структура сайта itsclear.ru
----------------------------------------
Уровни иерархии (сверху вниз):

  host  — корень хоста (apache), содержит map.xml, делящий сайт на разделы
  site  — подраздел (app/, ref/, resources/, map.xml, index.php)
  top   — группа справочников (папка в ref/), например it, school, web
  man   — справочник (manual), содержит app/, pages/, phpinc/, resources/
  cat   — категория страниц (папка в pages/)
  page  — отдельная страница (*.php)

Уровни описаны в Iclear.levels_list:
  ['host', 'site', 'top', 'man', 'cat', 'page']

2.1. Структура site:
  site/
  ├── app/
  │   ├── fonts/
  │   ├── scripts/
  │   └── styles/
  ├── ref/
  │   ├── topic_1/
  │   └── topic_2/
  ├── resources/
  ├── index.php
  └── map.xml

2.2. Структура man (справочник):
  man/
  ├── app/
  │   ├── css/
  │   └── scripts/
  ├── pages/
  │   ├── category_1/
  │   └── category_2/
  ├── phpinc/
  │   ├── aside.php
  │   ├── header.php
  │   └── parthead.php
  ├── resources/
  │   └── examples/
  ├── index.php
  └── map.xml

2.3. Файлы map.xml
-------------------
Файлы map.xml находятся на трёх уровнях: host, site, man. Каждый уровень
имеет свою структуру вложенности.

**Уровень host** — самый простой, одноуровневый список site-разделов.
Используется для формирования ссылок в корневом index.php.

**Уровень site** — навигатор по первому уровню разделов справочников.
Глубина вложенности — 2 уровня (top → man). 

**Уровень man** — навигатор по второму уровню, конечные ссылки —
страницы. Глубина вложенности — 2 уровня (cat → page).

Атрибуты элементов li (используются в iclearlib для навигации,
генерации index.php, aside.php и проверки целостности сайта):

  `data-name`   — Техническое имя (латиница, CamelCase)
  `data-fname`  — Полное английское название
  `data-id`     — Уникальный идентификатор для URL/якорей
  `data-path`   — Относительный путь к странице
  `data-runame` — Русскоязычное отображаемое имя

3. Модуль iclearlib.py
-----------------------
Классы:

3.1. ANode (базовый узел дерева)
  - nid (str) — уникальный идентификатор
  - parent, children — ссылки на родителя и список потомков
  - lvl (int) — уровень в дереве
  - set_parent(node), add_child(node), add_children(list)
  - find_child(nid) — поиск по всему поддереву
  - travers(func) — обход поддерева вниз с вызовом func
  - render(), render_info() — вывод дерева

3.2. Iclear (конфигурация, static)
  - levels_list = ['host', 'site', 'top', 'man', 'cat', 'page']
  - levels_index — словарь {имя_уровня: номер}
  - topdir = "ref", pagedir = "pages", phpincdir = "phpinc"
  - pageext = ".php", mapname = "map.xml"
  - is_level(level), num_level(level)
  - next_level(level), prev_level(level)
  - call_func_by_levels(func)

3.3. INode(ANode) — узел с атрибутами уровня
  - nid, level, name, fname, runame, path
  - info(), finfo()

3.4. IHost(ANode) — корень хоста
  - hostpath, sitepath
  - full_path() — объединение hostpath + sitepath
  - fill() — парсит map.xml, создаёт ISite-узлы

3.5. ISite(INode) — подраздел сайта
  - full_path(), fill_sites(), fill_mans()

3.6. ITop(INode) — группа справочников

3.7. IMan(INode) — справочник
  - short_path(), full_path()
  - fill() — парсит map.xml, создаёт ICat + IPage

3.8. ICat(INode) — категория страниц
  - short_path(), full_path()

3.9. IPage(INode) — страница
  - short_path(), full_path()

3.10. NodesNav(Iclear) — состояние навигации
  - host, site, top, man, cat, page (атрибуты-узлы)
  - set_dict(dict) — установка узлов по словарю {site, top, man, cat, page}
  - get_dict() — возвращает словарь
  - get_url() — генерация URL вида:
    http://host/site/ref/top/man/pages/cat/page.php
  - set_from_url(url) — парсинг URL в узлы
  - set_from_page_path(path) — парсинг файлового пути

3.11. Service — генераторы и утилиты
  - gen_plug_pagephp(page) — создаёт шаблон *.php страницы
  - gen_index_phpinc(man) — генерирует index.php + phpinc/*.php
  - gen_root_indexphp(site) — генерирует корневой index.php раздела
  - gen_indexphp(man) — index.php справочника
  - gen_asidephp(man) — aside.php (боковое меню)
  - gen_headerphp(man) — header.php
  - gen_partheadphp(man) — parthead.php
  - check_map(man) — проверка уникальности id и path в карте
  - diff_map_by_dir(man) — сверка map.xml с файловой структурой,
    переносит лишние файлы в pages/errtemp

3.12. Tools — утилиты
  - set_output(stdout, outhandler) — настройка вывода
  - output(text, type) — вывод с цветом (ошибки красным)
  - check_exists_file(fname) — проверка существования файла
  - write_file(fname, text) — запись файла

4. Модуль wiclear.py
---------------------
Виджеты PyQt5:

4.1. WinNode(QWidget) — селектор одного уровня
  - QLabel (название уровня) + QComboBox (список узлов)
  - fill_list() — заполнение списка из NodesNav
  - При изменении обновляет NodesNav и каскадно перезаполняет
    селекторы следующих уровней

4.2. WinNodesNav(QWidget) — панель навигации (site → top → man → cat → page)
  - Содержит 5 WinNode: wsite, wtop, wman, wcat, wpage
  - set_nodnav(level, node) — установка узла в NodesNav
  - refill() — перезаполнение всех списков

4.3. IclearWidget(QWidget) — основная панель инструментов
  - WinNodesNav + кнопки:
    - URL — ввод/редактирование URL (QInputDialog)
    - Open — открытие страницы в редакторе
    - Menu ("...") — выпадающее меню с операциями:
      * xdg_open — открытие в браузере
      * refill host — перезагрузка карты сайта
      * gen_index_and_phpinc — генерация index.php + phpinc
      * gen_plug_pagephp — создание шаблона страницы
      * gen_content_list — генерация списка содержимого
      * check_map — проверка map.xml

4.4. IclearSettingDialog(QDialog) — настройки
  - hostpath (apache sites dir)
  - sitepath (additional path)

5. Интеграция с ClearEditor
----------------------------
Пакет iclear планируется подключить как плагин ClearEditor.
Для этого необходимо:

  5.1. Создать manifest.json в папке iclear/
  5.2. Создать класс-наследник PluginBase, который в on_load()
       создаёт IclearWidget и добавляет его, например, в док-панель
       или тулбар.
  5.3. Адаптировать import в wiclear.py (замена относительного импорта
       на импорт из пакета плагина).

6. Соглашения
-------------
- Все идентификаторы (nid) уникальны в пределах своего уровня.
- PHP-файлы (кроме index.php и phpinc/) находятся только в pages/.
- Ресурсы — в resources/, стили/скрипты — в app/.
