4. ТРЕБОВАНИЯ К ФУНКЦИОНАЛЬНОСТИ ЗАГРУЗКИ ПЛАГИНОВ
-----------------
4.1. У каждого плагина в папке plugins есть своя папка и файл манифеста
(manifest.json)
   {
       "name": "wordcount",
       "description": "Подсчёт слов в редакторе",
       "entry": "main.py",
       "class": "WordCountPlugin"
   }
4.2. Обнаружение плагинов производится с помощью чтения их манифестов
Plugin Manager (класс PluginManager):
   - Сканирует plugins/
   - Читает manifest.json в каждой подпапке
   - Динамически импортирует модуль (importlib)
   - Отдаёт список плагинов в PluginWidget (QListView)
   - Активация -> on_load(editor), деактивация -> on_unload()
   - При активации создаёт UI-элементы (статус-бар, меню, тулбары, док-виджеты)
     согласно декларации плагина, при деактивации — удаляет их

4.3. Базовый класс для плагинов
(не используем ABCMeta)
class PluginBase(QObject):
    name: str
    description: str
    actions: list[dict] = []
    menu_items: list[dict] = []
    toolbar_items: list[dict] = []
    status_fields: dict[str, dict] = {}

    def on_load(self, editor: QTextEdit):
        """Вызывается при активации плагина."""

    def on_unload(self):
        """Вызывается при деактивации плагина."""

    def create_dock_widget(self, parent=None) -> QWidget | None:
        """Возвращает виджет для размещения в док-панели EditorApp."""
        return None

4.4. Выбор подключаемых плагинов — QListView с чекбоксами
   (QStandardItemModel, item.setCheckable(True)), отдельное окно,
   открывается в меню Settings.
   Отмеченные чекбоксом плагины будут подключены после нажатия OK.
4.5. Пользователь может подключить и отключить любое количество плагинов.
4.6. Между сессиями конфигурация плагинов сохраняется с помощью
   QSettings PyQt5, QSettings.IniFormat, настройки хранятся рядом с main.py,
4.7. При открытии основного приложения все ранее подключенные плагины стартуют.

4.8. Декларация UI-элементов плагина
4.8.1. Плагины могут влиять на UI EditorApp через атрибуты класса:
   - actions — плоский список действий (переиспользуемые декларации QAction)
   - menu_items — вложенный список для построения QMenu/QAction в menuBar
   - toolbar_items — вложенный список для построения QToolBar
   - status_fields — PluginManager создаёт QLabel в statusBar,
     подключает сигнал плагина к обновлению текста лейбла
   - create_dock_widget() — плагин возвращает QWidget,
     PluginManager оборачивает в QDockWidget и добавляет в EditorApp
   При деактивации плагина PluginManager удаляет все созданные элементы.
   EditorApp не содержит кода, зависящего от конкретных плагинов.

4.8.2. Cписок ключей декларации actions: id, text, kind, callback, icon,
   shortcut, statustip, tooltip.
   Ключ content — только для 'kind': "menu" (если kind=menu, а content
   отсутствует, считаем что content = []).
   Ключ "kind" может иметь значения "action" | "menu" | "separator".
   Если для kind:"action" callback не задан — оставляем без callback.

4.8.3. Индексация actions. Для переиспользования actions группируются
   в index_actions по полю id. Ключ kind:"action" проставляется на этапе
   индексации:

   index_actions = {}
   for i in actions:
       i["kind"] = "action"
       index_actions[i['id']] = i

4.8.4. Пример полной декларации плагина

class SamplePlugin(PluginBase):
    name = "sample"
    description = "Пример плагина"

    actions = [
        {'id': "replace_entity", 'text': "Replace entity"},
        {'id': "wrap_p", 'text': "Wrap <p>",
         'icon': 'icons/tagp.png', 'tooltip': "Wrap &lt;p&gt;",
         'statustip': "Wrap selected <p>...</p>"},
        {'id': "wrap_b", 'text': "Wrap <b>", 'icon': 'icons/tagb.png'},
        {'id': "make_list", 'text': "ul", 'tooltip': "Make list"},
        {'id': "remove_empty", 'text': "Remove empty lines",
         'callback': "remove_empty_lines_in_selected"},
        {'id': "capitalize_first", 'text': "Capitalize first letters",
         'callback': "capitalize_first_letters_in_selected"},
    ]

    index_actions = {}
    for i in actions:
        i["kind"] = "action"
        index_actions[i['id']] = i

    menu_items = [{
        'kind': "menu", 'text': "Text", 'content': [
            index_actions['remove_empty'],
            index_actions['capitalize_first'],
        ]}, {
        'kind': "menu", 'text': "HTML", 'content': [
            index_actions['replace_entity'],
            {'kind': "menu", 'text': "Wrap", 'content': [
                index_actions['wrap_p'],
                index_actions['wrap_b'],
            ]},
            {'kind': "separator"},
            index_actions['make_list'],
        ]},
    ]

    toolbar_items = [{
        'kind': "menu", 'text': "HTML", 'content': [
            index_actions['wrap_p'],
            index_actions['wrap_b'],
            index_actions['make_list'],
        ]},
    ]

    def on_load(self, editor):
        self._panel = editor.tab_panel

    def on_unload(self):
        self._panel = None

    def remove_empty_lines_in_selected(self):
        ...

    def capitalize_first_letters_in_selected(self):
        ...

    def replace_entity(self):
        ...

4.9. PluginManager — вспомогательные методы _setup_ui

4.9.1. Создание отдельного QAction (выделено в отдельную функцию):
   - create_action(self, menu, text, callback,
        icon=None, shortcut=None, statustip=None, tooltip=None)
     — создаёт QAction, подключает callback (если задан), устанавливает
       опциональные атрибуты. Возвращает QAction.

4.9.2. Построение меню/тулбара из вложенного списка (рекурсивно,
   подходит и для menu_items, и для toolbar_items):
   - create_ui_from_actions(self, target, actions, plugin)
     — рекурсивно строит UI из вложенного списка.
     - kind='menu':
       * target имеет addMenu (QMenu) → создаёт подменю
       * target — QToolBar → создаёт QToolButton с QMenu
     - kind='separator': addSeparator()
     - kind='action': create_action(), callback резолвится через getattr(plugin, ...)
     - Возвращает список кортежей (тип, объект) для последующей очистки.

4.9.3. create_ui_from_actions применяется как для menu_items (QMenu),
   так и для toolbar_items (QToolBar). Для тулбара kind='menu' создаёт
   QToolButton с QMenu (popup-кнопка). Импорт: QToolButton из PyQt5.QtWidgets.

4.9.4. _setup_ui:
     - status_fields: без изменений
     - menu_items: итерирует entry, для kind='menu' находит/создаёт QMenu,
       вызывает create_ui_from_actions, результат в menu_ui_objects
     - toolbar_items: создаёт QToolBar, вызывает create_ui_from_actions
       для всех entry, результат в menu_ui_objects
     - create_dock_widget: без изменений
   - deactivate: menu_ui_objects — итерация по (kind, obj), obj.deleteLater()

4.9.5. variant_tool_submenu — классовая переменная PluginManager:
   PluginManager.variant_tool_submenu = 'InstantPopupArrow'
   Варианты:
   - 'InstantPopupArrow': QToolButton.InstantPopup + символ ↓ в тексте
   - 'InstantPopup': QToolButton.InstantPopup, без символа
   - 'MenuButtonPopup': QToolButton.MenuButtonPopup (стрелка отдельно)
   Переключение без перезапуска — меняется поведение для новых плагинов.

4.9.6. Особенности реализации:
	- create_ui_from_actions принимает plugin для резолва callback'ов
	  по имени (getattr). Это позволяет хранить в actions строковые
	  имена методов и поддерживать единый механизм.
	- Для QToolBar kind='menu' создаёт QToolButton с QMenu.
	  Вложенные подменю (menu внутри menu тулбара) работают через
	  рекурсию: target становится QMenu (есть addMenu).
	- В deactivate toolbar удаляется целиком (removeToolBar + deleteLater),
	  все дочерние виджеты (QToolButton, QMenu) удаляются вместе с тулбаром.
	  Дополнительно они сохраняются в menu_ui_objects для единообразной
	  очистки.
