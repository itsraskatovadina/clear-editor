🏗️ ClearEditor: Architecture Overview

A practical guide to the project's architecture and its evolution.

About this document: This guide describes the architecture of ClearEditor — a multi-tab text editor built with Python and PyQt5. It is designed as a teaching resource for software architecture principles, showing both the initial "legacy" structure and the target "clean" architecture after refactoring.

1. Initial Architecture (Legacy)

📁 Project Structure (before)

clear-editor/
├── core/              # Core application components
│   ├── editor_app.py     # Main window (everything in one class)
│   ├── tab_panel.py      # Tab management
│   ├── file_edit.py      # File editor (mixed logic)
│   └── msg_panel.py      # Message panel
├── plugins/           # Plugins (wordcount, textprocessing, iclear)
├── plugins_service/   # Plugin infrastructure
├── tests/             # Manual visual tests
├── specifications/    # Russian-language requirements
├── main.py            # Entry point
└── config.json        # Settings (sessions, plugins)

🔍 Key Architectural Issues

IssueLocationWhy It's a Problem
Mixing ResponsibilitiesFileEdit contains file I/O logicViolates Single Responsibility Principle (SRP)
Tight CouplingEditorApp depends directly on all componentsHard to test and modify
Business Logic in GUIPlugins work directly with widgetsLogic cannot be reused without the UI
No Automated TestsOnly manual visual checksNo regression protection
Hard to ExtendNew features require changes to EditorAppViolates Open/Closed Principle (OCP)

2. Target Architecture (Refactored)

🎨 Core Principles

  • Layered Architecture — Presentation (GUI), Business Logic (Services), Data (Models)
  • Dependency Injection — components receive services via constructors
  • Single Responsibility — each class does one thing
  • Testability — business logic can be tested without PyQt

📁 New Structure

clear-editor/
├── core/              # Presentation Layer (GUI)
│   ├── main_window.py    # Coordination only
│   ├── tab_panel.py      # Tab management only
│   ├── file_edit.py      # Text display only
│   └── msg_panel.py      # Message display only
│
├── services/          # Business Logic Layer
│   ├── file_service.py     # Open, save, external change tracking
│   ├── session_service.py  # Session persistence
│   ├── plugin_service.py   # Plugin management
│   └── settings_service.py # Config.json handling
│
├── models/            # Data Layer
│   ├── file_state.py      # File state (dataclass)
│   └── session_state.py   # Session state
│
├── plugins/           # Plugins (use services)
├── tests/             # Automated tests (pytest + pytest-qt)
├── main.py
└── config.json

3. Key Code Transformations

Example 1: Saving a File

Before (core/file_edit.py):

def save_file(self):
    try:
        with open(self.file_path, 'w', encoding='utf-8') as f:
            f.write(self.toPlainText())
        self.document().setModified(False)
    except Exception as e:
        QMessageBox.critical(self, "Error", f"Failed to save: {e}")

After (services/file_service.py + core/file_edit.py):

# In service — business logic only
class FileService:
    def save_file(self, state: FileState) -> None:
        try:
            with open(state.file_path, 'w', encoding=state.encoding) as f:
                f.write(state.content)
            state.is_modified = False
        except Exception as e:
            raise IOError(f"Save failed: {e}")

# In view — coordination only
def save_current_tab(self):
    current_edit = self.tab_panel.current_widget()
    if current_edit:
        state = current_edit.get_state()
        try:
            self.file_service.save_file(state)
            current_edit.set_state(state)
        except IOError as e:
            self.msg_panel.show_error(str(e))

Example 2: Session Loading

Before (core/editor_app.py):

def load_session(self):
    with open('config.json', 'r') as f:
        config = json.load(f)
    for file_path in config.get('open_files', []):
        if os.path.exists(file_path):
            self.open_file(file_path)

After (services/session_service.py + core/main_window.py):

# In service
class SessionService:
    def load_session(self) -> List[Path]:
        config = self.settings_service.get_config()
        return [Path(p) for p in config.get('open_files', []) 
                if Path(p).exists()]

# In main window
def _restore_session(self):
    paths = self.session_service.load_session()
    for path in paths:
        self._on_tab_open_request(path)
