4. PLUGIN LOADING FUNCTIONALITY REQUIREMENTS
-----------------
4.1. Each plugin has its own folder under plugins/ and a manifest file
(manifest.json)
   {
       "name": "wordcount",
       "description": "Word count in editor",
       "entry": "main.py",
       "class": "WordCountPlugin"
   }
4.2. Plugin discovery is done by reading manifests
Plugin Manager:
   - Scans plugins/
   - Reads manifest.json in each subfolder
   - Dynamically imports the module (importlib)
   - Returns plugin list to PluginWidget (QListView)
   - Activation -> on_load(editor), deactivation -> on_unload()
   - On activation creates UI elements (status bar, menu, toolbars, dock widgets)
     according to plugin declaration, on deactivation — removes them

4.3. Base class for plugins
(does not use ABCMeta)
class PluginBase(QObject):
    name: str
    description: str
    actions: list[dict] = []
    menu_items: list[dict] = []
    toolbar_items: list[dict] = []
    status_fields: dict[str, dict] = {}

    def on_load(self, editor: QTextEdit):
        """Called when plugin is activated."""

    def on_unload(self):
        """Called when plugin is deactivated."""

    def create_dock_widget(self, parent=None) -> QWidget | None:
        """Returns a widget for placement in EditorApp dock panel."""
        return None

4.4. Plugin selection — QListView with checkboxes
   (QStandardItemModel, item.setCheckable(True)), separate window,
   opened from Settings menu.
   Checked plugins will be loaded after pressing OK.
4.5. User can enable and disable any number of plugins.
4.6. Between sessions, plugin configuration is saved using
   PyQt5 QSettings with QSettings.IniFormat, settings stored next to main.py.
4.7. On application startup, all previously loaded plugins are activated.

4.8. Plugin UI declaration
4.8.1. Plugins can affect EditorApp UI through class attributes:
   - actions — flat list of actions (reusable QAction declarations)
   - menu_items — nested list for building QMenu/QAction in menuBar
   - toolbar_items — nested list for building QToolBar
   - status_fields — PluginManager creates QLabel in statusBar,
     connects plugin signal to label text update
   - create_dock_widget() — plugin returns QWidget,
     PluginManager wraps it in QDockWidget and adds to EditorApp
   On plugin deactivation, PluginManager removes all created elements.
   EditorApp does not contain code dependent on specific plugins.

4.8.2. Action declaration keys: id, text, kind, callback, icon,
   shortcut, statustip, tooltip.
   Key content — only for 'kind': "menu" (if kind=menu and content
   is absent, content is considered []).
   Key "kind" can have values "action" | "menu" | "separator".
   If kind:"action" has no callback — leave without callback.

4.8.3. Actions indexing. For reuse, actions are grouped into
   index_actions by id. The kind:"action" key is set during indexing:

   index_actions = {}
   for i in actions:
       i["kind"] = "action"
       index_actions[i['id']] = i

4.8.4. Complete plugin declaration example

class SamplePlugin(PluginBase):
    name = "sample"
    description = "Sample plugin"

    actions = [
        {'id': "replace_entity", 'text': "Replace entity"},
        {'id': "wrap_p", 'text': "Wrap <p>",
         'icon': 'icons/tagp.png', 'tooltip': "Wrap &lt;p&gt;",
         'statustip': "Wrap selected <p>...</p>"},
        {'id': "wrap_b", 'text': "Wrap <b>", 'icon': 'icons/tagb.png'},
        {'id': "make_list", 'text': "ul", 'tooltip': "Make list"},
        {'id': "remove_empty", 'text': "Remove empty lines",
         'callback': "remove_empty_lines_in_selected"},
        {'id': "capitalize_first", 'text': "Capitalize first letters",
         'callback': "capitalize_first_letters_in_selected"},
    ]

    index_actions = {}
    for i in actions:
        i["kind"] = "action"
        index_actions[i['id']] = i

    menu_items = [{
        'kind': "menu", 'text': "Text", 'content': [
            index_actions['remove_empty'],
            index_actions['capitalize_first'],
        ]}, {
        'kind': "menu", 'text': "HTML", 'content': [
            index_actions['replace_entity'],
            {'kind': "menu", 'text': "Wrap", 'content': [
                index_actions['wrap_p'],
                index_actions['wrap_b'],
            ]},
            {'kind': "separator"},
            index_actions['make_list'],
        ]},
    ]

    toolbar_items = [{
        'kind': "menu", 'text': "HTML", 'content': [
            index_actions['wrap_p'],
            index_actions['wrap_b'],
            index_actions['make_list'],
        ]},
    ]

    def on_load(self, editor):
        self._panel = editor.tab_panel

    def on_unload(self):
        self._panel = None

    def remove_empty_lines_in_selected(self):
        ...

    def capitalize_first_letters_in_selected(self):
        ...

    def replace_entity(self):
        ...

4.9. PluginManager — _setup_ui helper methods

4.9.1. Creating a standalone QAction (extracted into a separate function):
   - create_action(self, menu, text, callback,
        icon=None, shortcut=None, statustip=None, tooltip=None)
      — creates QAction, connects callback (if given), sets optional
        attributes. Returns QAction.

4.9.2. Building menu/toolbar from a nested list (recursively,
   works for both menu_items and toolbar_items):
   - create_ui_from_actions(self, target, actions, plugin)
      — recursively builds UI from a nested list.
      - kind='menu':
        * target has addMenu (QMenu) → creates submenu
        * target is QToolBar → creates QToolButton with QMenu
      - kind='separator': addSeparator()
      - kind='action': create_action(), callback resolved via getattr(plugin, ...)
      - Returns a list of (type, object) tuples for cleanup.

4.9.3. create_ui_from_actions is used for both menu_items (QMenu)
   and toolbar_items (QToolBar). For toolbar kind='menu' creates
   a QToolButton with QMenu (popup button). Import: QToolButton from
   PyQt5.QtWidgets.

4.9.4. _setup_ui:
       - status_fields: unchanged
       - menu_items: iterates entries, for kind='menu' finds/creates QMenu,
         calls create_ui_from_actions, appends to menu_ui_objects
       - toolbar_items: creates QToolBar, calls create_ui_from_actions
         for all entries, appends to menu_ui_objects
       - create_dock_widget: unchanged
   - deactivate: menu_ui_objects — iterates (kind, obj), obj.deleteLater()

4.9.5. variant_tool_submenu — PluginManager class variable:
   PluginManager.variant_tool_submenu = 'InstantPopupArrow'
   Options:
   - 'InstantPopupArrow': QToolButton.InstantPopup + ↓ symbol in text
   - 'InstantPopup': QToolButton.InstantPopup, no symbol
   - 'MenuButtonPopup': QToolButton.MenuButtonPopup (separate arrow zone)
   Can be changed at runtime (affects newly activated plugins).

4.9.6. Implementation notes:
  	- create_ui_from_actions accepts plugin for callback resolution
  	  by name (getattr). This allows storing string method names
  	  in actions and maintains a single mechanism.
  	- For QToolBar kind='menu' creates QToolButton with QMenu.
  	  Nested submenus (menu inside toolbar menu) work via recursion:
  	  target becomes QMenu (has addMenu).
  	- In deactivate, the toolbar is removed entirely (removeToolBar
  	  + deleteLater). Child widgets (QToolButton, QMenu) are deleted
  	  with the toolbar. They are also stored in menu_ui_objects for
  	  uniform cleanup.
