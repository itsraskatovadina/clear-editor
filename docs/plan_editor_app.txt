План этапа 4 — EditorApp: вынос Config/Session/Zoom сервисов
=============================================================

Текущая проблема: EditorApp (~160 строк) удерживает 3 ответственности:
1. Загрузка/сохранение config.json, QSettings (geometry, font, recent_files, open_files)
2. Управление зумом (zoom_in/zoom_out)
3. Восстановление open_files при старте (set_tab_panel)

Попутный баг: в текущем коде `set_tab_panel` определён дважды —
новый (строки 94-103) переопределён старым (строки 162-176, читает QSettings напрямую).
Старый вариант надо удалить.

Предложение по следующему этапу: iclear plugin
==============================================

### Проблема

Плагин iclear использует старое API `editor.tab_panel`, которого больше нет:

- `main.py` строка 17: `self._editor.tab_panel.tab_added` — сигнал не существует
- `main.py` строка 26: `self._editor.tab_panel.currentChanged` → `file_tab_view.currentChanged`
- `main.py` строка 30: `self._editor.tab_panel.widget(index)` → виджеты больше
  не имеют `.path` (теперь это EditorWidget без path)
- `wiclear.py` строка 255: `self.parent.tab_panel.add_tab(Path(...))` → add_tab
  теперь в FileTabSrv

### Варианты решений

**A. Через editor_state_changed (рекомендуется)**

FileTabSrv испускает `editor_state_changed(title, full_title, mod_label, operation)`
при смене/открытии/закрытии вкладки. `full_title` = path файла.

Плагин подключается к сигналу, фильтрует `operation in ("tab_changed", "Opened", "Reload")`,
передаёт `full_title` в `IclearWidget.fill_from_page_path()`.

Для open_page: передать в IclearWidget колбэк `editor.file_tab_srv.add_tab`.

Плюсы: не требует новых методов, минимум изменений.
Минусы: нет доступа к документу по произвольному индексу (iclear и так работает
только с текущей вкладкой).

**B. С новым методом FileTabSrv.document_at(index)**

Добавить в FileTabSrv публичный метод `document_at(index: int) → Optional[Document]`,
который пробрасывает `_model.at(index)`.

Плагин подключается к `file_tab_view.currentChanged[int]` и получает Document
по индексу, из которого берёт `file_path`.

Плюсы: прямой доступ к любому документу.
Минусы: изменение FileTabSrv, дополнительный публичный метод.

### Что изменить в плагине iclear

1. **main.py**:
   - `on_unload`: `tab_panel.tab_added.disconnect` → убрать мёртвый код
   - `create_toolbar_widget`: `tab_panel.currentChanged` → `file_tab_srv.editor_state_changed`
   - `_on_current_tab_changed(index)` → `_on_tab_changed(title, full_title, ...)`
   - добавить `_sync_from_current_tab()` для инициализации при старте
   - передать колбэк editor.file_tab_srv.add_tab в IclearWidget

2. **wiclear.py — IclearWidget**:
   - принять  `open_file_cb` в конструкторе
   - `open_page`: `self.parent.tab_panel.add_tab` → `self._open_file_cb(path)`

3. **Проверка**:
   - после рефакторинга — ручная проверка:
     - iclear тулбар отображается
     - навигация по дереву работает
     - открытие страницы из iclear открывает вкладку
     - при переключении вкладок дерево синхронизируется
