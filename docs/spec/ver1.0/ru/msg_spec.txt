ТРЕБОВАНИЯ К ФУНКЦИОНАЛЬНОСТИ MsgPanel и ErrorHandler
(specifications/msg_spec.txt)

Панель сообщений и перехватчик stderr.

1. MsgPanel(QTabWidget) — панель сообщений в нижней части EditorApp.
   Содержит две вкладки: Err (ошибки) и Msg (сообщения).
   Вкладки расположены слева (QTabWidget.West).
   Оба текстовых поля только для чтения (setReadOnly(True)).
   Включает прокрутку кнопками (setUsesScrollButtons(True)).
   При запуске активна вкладка Err (setCurrentIndex(0)).

2. СИГНАЛЫ
   - new_error = pyqtSignal(str) — для критических ошибок из stderr.

3. АТРИБУТЫ ЭКЗЕМПЛЯРА
   - err_box: QTextEdit — текстовое поле для ошибок (вкладка "Err").
   - msg_box: QTextEdit — текстовое поле для сообщений (вкладка "Msg").
   - app: QApplication — ссылка на экземпляр приложения (для sound beep).

4. КОНСТРУКТОР
   def __init__(self, parent=None):
   - Создаёт err_box и msg_box (QTextEdit, только чтение).
   - Добавляет вкладки: addTab(err_box, "Err"), addTab(msg_box, "Msg").
   - setTabPosition(QTabWidget.West).
   - setUsesScrollButtons(True).
   - setCurrentIndex(0) — активна вкладка Err.
   - Получает self.app = QApplication.instance().
   - Подключает сигнал: new_error → self._display_error.

5. МЕТОДЫ

   5.1. Отображение критических ошибок (из stderr):
        def _display_error(self, msg):
        - Добавляет в err_box текст красного цвета (HTML <span style='color: red;'>).
        - Переключает видимую вкладку на err_box.
        - Воспроизводит звуковой сигнал (self.app.beep()).

   5.2. Отображение сообщений:
        def new_message(self, msg, sender=None, msgtype=None):
        - Цвет зависит от msgtype:
            'debug' → gray, 'info' → blue, 'warning' → orange, 'error' → red.
          По умолчанию (msgtype=None) — blue.
        - Если передан sender, добавляется префикс '[sender] '.
        - Выводит сообщение в msg_box как HTML-строку.
        - Переключает видимую вкладку на msg_box.
        - Воспроизводит звуковой сигнал (self.app.beep()).

6. ErrorHandler — замена sys.stderr для перенаправления ошибок
   в EditorApp.on_error. Duck-typed как файловый объект (write/flush).

   АТРИБУТЫ ЭКЗЕМПЛЯРА
   - original_stderr: TextIO — оригинальный sys.stderr.
   - handle: callable — функция обратного вызова (EditorApp.on_error).

   МЕТОДЫ
   def __init__(self, handle):
   - Сохраняет sys.stderr в self.original_stderr.
   - Сохраняет handle (EditorApp.on_error).

   def write(self, message):
   - Пишет в original_stderr.
   - Вызывает handle(message).

   def flush(self):
   - Вызывает original_stderr.flush().

7. ЦЕПОЧКА СООБЩЕНИЙ
   FileEditor.message → TabPanel (проброс) → EditorApp.on_message
     → MsgPanel.new_message(msg, sender, msgtype)
   sys.stderr → ErrorHandler.write → EditorApp.on_error
     → MsgPanel.new_error.emit → MsgPanel._display_error
