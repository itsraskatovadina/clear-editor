6. ТРЕБОВАНИЯ К ФУНКЦИОНАЛЬНОСТИ TabPanel (editor_panel_spec.txt)

Контейнер документов с вкладками, управляющий несколькими экземплярами FileEditor

6.1. TabPanel — QTabWidget, окно редактора мультидокументов в основном классе EditorApp,
каждая вкладка содержит FileEditor. Он управляет жизненным циклом вкладок (открытие,
закрытие, сохранение, перезагрузка, изменение порядка), отслеживает изменения на разных вкладках,
   обнаруживает изменения во внешнем файле и отправляет агрегированные сигналы состояния
   в родительское главное окно или контроллер.

6.2. СИГНАЛЫ
     - tab_added = pyqtSignal(object) — emitted в add_tab() после добавления вкладки,
       передаёт внутренний редактор (QTextEdit/ExtEditor).
     - editor_state_changed = pyqtSignal(str, str, str, str)
       Параметры: (file_name, full_path, modification_label, operation)
       operation: 'tab_changed', 'modification_changed', 'Opened', 'New', 'Saved', 'Saved as', 'Reload', 'Reopened'
     - line_changed = pyqtSignal(int)
       Параметр: номер строки (1-based).
     - message = pyqtSignal(str, str, str)
       Параметры: (message_text, sender, message_type)

6.3. Вкладки можно перетаскивать (setMovable(True)) и закрывать (setTabsClosable(True)).

6.4. Нельзя закрыть последнюю оставшуюся вкладку (close_tab проверяет self.count() > 1), кроме особых случаев.

6.5. АТРИБУТЫ ЭКЗЕМПЛЯРА
     - do_not_process_the_tab_change: bool — флаг подавления обработки переключения вкладок.
     - editor_class: type — класс редактора (по умолчанию QTextEdit).

6.6. КЛАСС: TabPanel(QTabWidget)
     def __init__(self, parent=None, editor_class=QTextEdit):
         super().__init__(parent)
         self.editor_class = editor_class
     - editor_class передается каждому FileEditor в новой вкладке.
     - Подключение: currentChanged.connect(self.on_current_tab_changed)
     - Начальная вкладка НЕ создаётся (self.add_tab() удалён). Первая вкладка
       создаётся через set_files() или EditorApp.set_tab_panel().

6.7. API

     6.7.1. Навигация:
            - current_editor() -> QTextEdit | None — возвращает активный редактор текущей вкладки.
            - get_current_editor() -> QWidget — возвращает текущий виджет (FileEditor).
            - get_current_path() -> str: путь к файлу текущего редактора или ''.

     6.7.2. Управление вкладками:
            - add_tab(path=None) -> bool:
              Если path — проверка, не открыт ли уже файл (если да — переключение на вкладку).
              Если path не существует — вызывает check_path_exists(), которая показывает
              диалог и возвращает False.
              Создаёт FileEditor, подключает его сигналы, добавляет вкладку.
              Если path задан — загружает файл через editor.load_from_file().
              Излучает editor_state_changed с операцией 'Opened' или 'New'.
            - new_tab(): wrapper для add_tab(None).
            - open_file(): QFileDialog.getOpenFileName → add_tab(Path(file_path)).
            - set_files(file_list: list[str]) -> int:
              Принимает список путей к файлам.
              Для каждого пути проверяет path.exists(), для существующих вызывает add_tab.
              Возвращает количество успешно открытых файлов.
            - close_tab(index, close_last_tab = False) -> bool:
              Последнюю вкладку можно закрыть только если это разрешено.
              Если документ изменён — диалог Save/Ignore/Cancel.
              Если файл внешне изменён — предупреждение.
              При Cancel возвращает False.
            - close_all_tab() -> bool: 
              Закрывает все вкладки в обратном порядке с вызовом close_tab(), разрешая закрывать последнюю вкладку.
            - current_tab_close() -> bool: закрывает текущую вкладку.

     6.7.3. Сохранение/загрузка:
            - save_tab(index, editor) -> bool: сохраняет файл. Если path=None — save_as_tab.
            - save_as_tab(index, editor) -> bool: QFileDialog.getSaveFileName, сохраняет.
            - current_tab_save(): сохраняет текущую вкладку.
            - current_tab_save_as(): сохраняет как текущую вкладку.
            - current_tab_reload(): перезагружает файл с диска, сохраняет позицию курсора.

     6.7.4. Информация:
            - current_tab_view_property(): QMessageBox с путём к файлу.
            - get_open_files() -> list[str]: список путей открытых файлов.
            - check_path_exists(path) -> bool: проверка существования файла, предупреждение.
            - set_tab_text_and_tooltip(name, fname, mod_label, index): обновляет текст и подсказку вкладки.

     6.7.5. Обработчики сигналов:
            - on_current_tab_changed(index): фокусирует редактор, излучает editor_state_changed.
            - on_user_modification_changed(name, fname, mod_label): обновляет текст вкладки,
              излучает editor_state_changed.
            - on_app_focus_changed(old, now): проверяет внешние изменения при возврате фокуса.
            - check_tab_externally_modified(index, editor): диалог Reload/Save/Ignore при внешнем изменении.

     6.7.6. closeEvent(event): вызывает close_all_tab(), при успехе — event.accept().

6.8. Пункты меню New и Close для TabPanel находятся в меню File EditorApp
     (см. editor_app_spec.txt).

6.9. Плагины через plugin_manager получают editor_widget = editor.current_editor().
     Плагин работает с текущим (активным) редактором независимо от того, на какой
     вкладке он находится. При переключении вкладок плагин продолжает работать
     с новым активным редактором (подписка на сигнал currentChanged TabPanel).
     Для подключения к новым вкладкам плагины подписываются на сигнал tab_added.
